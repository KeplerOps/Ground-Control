### Security
- Bound the mixed-graph API surface against unbounded traversal, path explosion, and projection-size DoS. `GraphNeighborhoodQueryRequest` and `GraphPathsQueryRequest` enforce per-field bean validation (depth `1..MAX`, root-node and entity-type list sizes, identifier lengths). `MixedGraphService` re-validates the same bounds at the service entry so internal callers cannot bypass HTTP-layer checks, maps unknown / blank `entityTypes` to a 422 validation envelope instead of a 500, and corrects an off-by-one that allowed `findPaths` to return a path with more edges than the caller requested. The `MixedGraphClient` contract now accepts an `entityTypes` filter so the AGE adapter applies the narrowing inside Cypher (parameter-bound `WHERE n.entity_type IN $entity_types`) and the `LIMIT (MAX + 1)` cap fires on the *filtered* set — a caller's narrowing actually narrows what the database materializes. The AGE-disabled fallback mirrors the same predicate in memory. AGE adapter constants delegate to a single canonical `GraphTraversalLimits` policy class so service, DTOs, and adapter cannot drift apart. `traverse()` now delegates to `extractSubgraph()` rather than duplicating the body. Internal comments cleaned up to satisfy static analysis.

### Security
- Close arbitrary-local-file exfiltration on the MCP upload boundary. `mcp/ground-control/lib.js` previously routed `gc_admin import_strictdoc`, `import_reqif`, and `pack-registry/import` through a trusted-arbitrary-path helper that fed any operator-supplied absolute path to `readFileSync()` and POSTed the bytes to the backend — a prompt-injected agent could read SSH keys, env files, or shell history and ship them to whatever `GC_BASE_URL` pointed at, bypassed only by the backend's `ROLE_ADMIN` check (which doesn't protect local files when the operator is the legitimate admin or the backend is remote). The hardened resolver `readApprovedUploadFile` enforces, in order: non-empty absolute path with no NUL bytes; lowercase extension match against a per-action allowlist (`.sdoc` / `.reqif` / `.json`); `lstat`-based leaf-symlink rejection before realpath; `realpath` canonicalization rooted at `process.cwd()` for containment so ancestor symlinks cannot escape the workspace; final `stat` to refuse directories, FIFOs, devices, and sockets; and a `readFileSync` of the canonical path so the bytes that ship match the path the validator approved. All three upload actions now route through it.
